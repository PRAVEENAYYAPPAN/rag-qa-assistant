Machine Learning and AI Knowledge Base
=======================================

1. INTRODUCTION TO MACHINE LEARNING
Machine learning (ML) is a subset of artificial intelligence that provides systems the ability to automatically learn and improve from experience without being explicitly programmed. ML focuses on the development of computer programs that can access data and use it to learn for themselves.

The process begins with observations or data, such as examples, direct experience, or instruction, to look for patterns in data and make better decisions in the future. The primary aim is to allow computers to learn automatically without human intervention.

2. TYPES OF MACHINE LEARNING
There are three main types of machine learning:

a) Supervised Learning: The algorithm is trained on labelled data. The model learns a mapping from inputs to outputs. Examples include linear regression, decision trees, support vector machines, and neural networks.

b) Unsupervised Learning: The algorithm works on unlabelled data, discovering hidden patterns without pre-existing labels. Examples include k-means clustering, hierarchical clustering, and autoencoders.

c) Reinforcement Learning: An agent learns to make decisions by taking actions in an environment to maximise cumulative reward. Examples include Q-learning, Deep Q-Networks (DQN), and Proximal Policy Optimisation (PPO).

3. NEURAL NETWORKS
A neural network is a series of algorithms that attempt to recognise underlying relationships in a set of data through a process that mimics the way the human brain operates. Neural networks consist of:

- Input Layer: Receives raw input features
- Hidden Layers: Transform inputs through learned weights and activation functions
- Output Layer: Produces predictions or classifications

Key concepts:
- Activation functions: ReLU, Sigmoid, Tanh, Softmax
- Backpropagation: Algorithm for training through gradient descent
- Weights and biases: Parameters learned during training
- Batch normalisation: Technique to stabilise and accelerate training

4. DEEP LEARNING
Deep learning is a subset of machine learning that uses neural networks with many layers (deep architectures). It excels at learning representations from raw data. Key architectures include:

- Convolutional Neural Networks (CNN): Optimised for image data
- Recurrent Neural Networks (RNN): Designed for sequential data
- Long Short-Term Memory (LSTM): Overcomes vanishing gradient in RNNs
- Transformers: Attention-based models revolutionising NLP and beyond

5. TRANSFORMERS AND ATTENTION MECHANISM
The Transformer architecture, introduced in "Attention Is All You Need" (Vaswani et al., 2017), relies entirely on an attention mechanism to draw global dependencies between input and output. Key components:

- Self-attention: Allows each position in a sequence to attend to all positions
- Multi-head attention: Runs attention multiple times in parallel
- Positional encoding: Injects sequence order information
- Feed-forward networks: Applied position-wise after attention layers

6. LARGE LANGUAGE MODELS (LLMs)
Large Language Models are deep learning models trained on vast amounts of text data. They can generate human-like text, answer questions, translate languages, and perform many other NLP tasks. Examples include GPT-4, Claude, Llama, Mistral, and Gemini.

LLMs are trained using self-supervised learning on massive corpora, then fine-tuned using Reinforcement Learning from Human Feedback (RLHF) to align with human preferences.

7. RETRIEVAL-AUGMENTED GENERATION (RAG)
RAG is a technique that enhances LLM responses by retrieving relevant context from a knowledge base before generating an answer. The pipeline includes:

a) Indexing Phase:
   - Documents are chunked into smaller passages
   - Each passage is encoded into a dense vector (embedding)
   - Embeddings are stored in a vector database (ChromaDB, Pinecone, FAISS)

b) Retrieval Phase:
   - User query is encoded into an embedding
   - Nearest neighbours are retrieved via cosine similarity or inner product
   - Re-ranking models improve precision of retrieved passages

c) Generation Phase:
   - Retrieved passages are injected into the LLM prompt as context
   - LLM generates a grounded, factual answer

Benefits of RAG:
   - Reduces hallucinations
   - Enables knowledge cutoff extension without retraining
   - Provides source citation and transparency

8. VECTOR DATABASES
Vector databases store high-dimensional embedding vectors and enable efficient approximate nearest-neighbour (ANN) search. Popular options:

- FAISS (Facebook AI Similarity Search): In-memory, high performance
- ChromaDB: Open-source, persistent, developer-friendly
- Pinecone: Managed cloud service
- Weaviate: Open-source with hybrid search
- Qdrant: Rust-based, high performance

9. EMBEDDING MODELS
Embedding models convert text into dense numerical vectors. Properties:
- Semantic similarity: Similar texts have similar embeddings
- Fixed dimensionality: all-MiniLM-L6-v2 produces 384-dim vectors
- Transfer learning: Pre-trained on massive corpora

Popular models:
- all-MiniLM-L6-v2: Fast, 384-dim, great balance
- all-mpnet-base-v2: High quality, 768-dim
- OpenAI text-embedding-ada-002: 1536-dim, strong performance
- E5, BGE, GTE: Recent high-performing open-source alternatives

10. EVALUATION METRICS IN INFORMATION RETRIEVAL
Measuring retrieval quality is critical for RAG systems:

- Precision@K: Fraction of top-K retrieved documents that are relevant
- Recall@K: Fraction of all relevant documents found in top-K results
- F1@K: Harmonic mean of precision and recall at K
- Mean Reciprocal Rank (MRR): Reciprocal of rank of first relevant result, averaged
- NDCG (Normalised Discounted Cumulative Gain): Measures ranking quality with position discount
- Hit Rate: Whether at least one relevant document appears in top-K

11. OPTIMISATION TECHNIQUES
Key techniques for improving RAG systems:

a) Chunking Strategy: Optimal chunk size balances context granularity vs. noise
b) Re-ranking: Cross-encoders score query-passage pairs more accurately than bi-encoders
c) Hybrid Search: Combining dense (semantic) + sparse (BM25) retrieval
d) Query Expansion: Generating multiple query variants to improve recall
e) HyDE (Hypothetical Document Embeddings): Generate a hypothetical answer and use it as query

12. FASTAPI FOR ML SERVICES
FastAPI is a modern, fast Python web framework ideal for ML model serving:
- Automatic OpenAPI documentation
- Pydantic validation of request/response models
- Async support with ASGI (uvicorn)
- Dependency injection for clean service architecture
- High performance comparable to Node.js and Go
